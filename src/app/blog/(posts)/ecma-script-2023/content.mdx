## New Features

### `Array.toReversed()`, `Array.toSorted()`, `Array.toSpliced()`

Immutable data in **<InternalLink href="/blog/ecma-script-2023">ECMAScript 2023</InternalLink>** ensures safer and more predictable code, reducing bugs and simplifying debugging processes. When using methods like `reverse`,`sort`, and `split`, they mutate your array. 🛠️ On the other hand, `toReversed()`, `toSorted()`, and `toSplit()` return a new array without mutating your original array.

<CodeHeader text='Example' />

```js
const originalArray = [4, 5, 1, 3, 2];

const sorted = originalArray.toSorted();
// sorted -> [1,2,3,4,5]
// originalArray -> [4,5,1,3,2]

const sliced = originalArray.toSpliced(0, 1);
// sliced -> [2,3,4,5]
// originalArray -> [4,5,1,3,2]

const reversed = originalArray.toReversed();
// reversed -> [2,3,1,5,4]
// originalArray -> [4,5,1,3,2]
```

### `Array.findLast()` and `Array.findLastIndex()`

Before ES14, to find the last element or index in an array that meets a condition, you typically had to reverse the array first. 🔄 The `findLast()` method retrieves the last matching element from an array. If no match is found, it returns undefined. 🔍 The `findLastIndex()` method operates similarly to findLast(), but instead of returning the element, it provides the index of the matching element.

<CodeHeader text='Example' />

```js
const originalArray = [105, 12, 84, 33, 25, 79, 55, 21];

const findLast = originalArray.findLast((item) => item > 60);
// findLast -> 79
// originalArray -> [105, 12, 84, 33, 25, 79, 55, 21]

const findLastIndex = originalArray.findLastIndexIndex((item) => item < 30);
// findLastIndex -> 7
// originalArray -> [105, 12, 84, 33, 25, 79, 55, 21]
```

### `Array.with()`

The `with()` method simplifies modifying a single element by index, providing a new array. 🔄 If you have both the index and the new value, `with()` streamlines the process.

<CodeHeader text='Example' />

```js
const originalArray = ["foo", "bar", "apple", "orange"];

originalArray.with(2, "banana");
// originalArray -> ['foo', 'bar', 'banana', 'orange']
```

### `Object.groupBy`

Suppose you have an array of objects and need to categorize them by property value, type, or quantity.

<CodeHeader text='Example' />

```js
const data = [
  { name: "apple", quantity: 84 },
  { name: "banana", quantity: 31 },
  { name: "orange", quantity: 6 },
  { name: "pich", quantity: 79 },
  { name: "avocados", quantity: 52 },
  { name: "olives", quantity: 3 },
];

function groupByCallback({ quantity }) {
  return quantity > 10 ? "normal" : "low";
}

const result = Object.groupBy(data, groupByCallback);
/** result ->
{
    "normal": [
        { "name": "apple","quantity": 84 },
        { "name": "banana","quantity": 31 },
        { "name": "pich","quantity": 79 },
        { "name": "avocados","quantity": 52 }
    ],
    "low": [
        { "name": "orange","quantity": 6 },
        { "name": "olives","quantity": 3}
    ]
}
**/
```

### #! JavaScript Shebang

In **<InternalLink href="/blog/ecma-script-2023">ECMAScript 2023</InternalLink>**, the introduction of Hashbang Grammar allows the use of shebang notation (#!) in source code. 📝 This feature helps differentiate between scripts and modules, improves compatibility with build tools, and aligns JavaScript with other languages.

you can run scripts by simply typing `./hello.js` in your shell. 📝 Without the #!, this wouldn't work.

<CodeHeader text='main.js' />

```js
#!/usr/bin/env node

console.log("Hello, world!");
// originalArray -> ['foo', 'bar', 'banana', 'orange']
```

<br />

<CodeHeader text='Terminal' />

```sh
./main.js
Hello, world!
```
